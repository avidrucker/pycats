# PM Phase 0 — data schema + hierarchical/parallel statechart (implementation plan)

## Context

pycats' north star is to become progressively faithful to **Super Smash Bros.
Project M**, intentionally adding mechanics not currently possible (see
`docs/research/pm-mechanics-implementation-analysis.md` and the project memory
`pycats-pm-fidelity-north-star`). **Phase 0** builds the foundation everything
else parameterizes off — the move/hitbox **data schema** and the fighter
statechart's **region decomposition** — and takes the one irreversible step the
roadmap depends on (circle geometry + the parity-oracle flip).

Approved design: `docs/superpowers/specs/2026-06-21-pm-phase0-schema-regions-design.md`.
Phase 0 adds no new *gameplay* mechanic; it restructures so later phases (real
knockback, hitstun, shieldstun, movesets, grabs, ledges…) are cheap additions.

## Architecture decisions (locked for this plan)

1. **Statechart becomes the primary engine.** `LegacyEngine` stays *constructible*
   as a coarse benchmark baseline, but it is no longer expected to match gameplay
   (the shared `Player.update`/`combat` change under it). Default backend for the
   game/runner flips to `statechart`.
2. **Cross-backend byte-identical parity is retired**, replaced by **statechart
   golden-snapshot** regression (same headless runner/harness; oracle changes from
   vs-legacy to vs-golden). The existing byte-identical tests
   (`tests/test_parity.py`, `tests/test_full_match.py`) are converted to golden
   tests; `tests/test_fighter_chart.py` legacy/statechart unit checks stay.
3. **Side effects are host-driven, not chart-driven (for now).** The chart owns
   the *phase labels* (`attacking.startup/active/recovery`); `Player.update` owns
   *spawning* the hitbox when the move enters its active window. This avoids
   statecharts-py's silent entry-action exception swallowing (confirmed in
   exploration). Migrating spawns into chart entry actions is a deliberate later
   option.
4. **Circle geometry for combat; rects untouched for terrain.** New
   `pycats/combat/geometry.py` overlap helpers; `combat.process_hits` switches
   from `colliderect` to circle-vs-circle-list.
5. **`defensive_status` ≈ existing `player.invulnerable`.** The new region
   formalizes the concept (`vulnerable`/`intangible`/`armored` placeholder); Phase 0
   maps `intangible ⇔ player.invulnerable` (already set by dodge, already read by
   `combat.process_hits`). No new timer needed.
6. **Schema is the Phase-0 subset only** (decision: every field used). `Hitbox =
   {circle, damage, angle}`; `base_knockback`/`knockback_growth` are deferred to
   Phase 1 with the real knockback formula. Phase 0 keeps the existing knockback
   constants/formula unchanged.

## Tasks (ordered, TDD; game stays runnable after each)

### 1. Move/hitbox data schema + load seam + first character
- Create `pycats/combat/__init__.py`, `pycats/combat/data.py` with frozen
  dataclasses `Circle`, `Hitbox{circle,damage,angle}`, `MoveData{name,in_air,
  startup,active,recovery,hitboxes}`, `Hurtbox{circles}`, `FighterData{hurtbox,
  moves}`, and `load_fighter_data(character) -> FighterData` (the single
  swap-to-files seam).
- Create `pycats/characters/__init__.py` + `pycats/characters/<name>.py` defining
  one `FighterData`: a 2-circle body hurtbox (approximating the 40×60 body) and one
  ground move `"attack"` whose `startup/active/recovery` and circle/damage/angle
  reproduce today's basic attack closely (≈ active beside the body, damage 10).
- Tests `tests/test_combat_data.py`: load returns expected `FighterData`; frozen;
  integer-frame fields. **No behavior change yet.**

### 2. Circle geometry helpers
- `pycats/combat/geometry.py`: `circle_overlap(c1, c2)` and
  `circles_overlap(circle, circle_list)` operating on absolute centers/radii, plus
  a helper to resolve a facing-relative `Circle` (dx,dy,r) to an absolute center
  from a player rect + facing (mirror dx when facing left).
- Tests `tests/test_geometry.py`: overlap/non-overlap, touching edge, facing-mirror.

### 3. Hierarchical + parallel fighter chart
- Rework `pycats/statecharts/fighter_chart.py` into the Section-2 tree: region
  `action` (`actionable{grounded{idle,run,shield}, airborne{jump,fall}}`,
  `attacking{startup,active,recovery}`, `dodging`, `hitstun{hurt}`, `ko`) +
  parallel region `defensive_status{vulnerable,intangible}`. Keep all transitions
  on the explicit `"tick"` event (no eventless); guards close over the live Player
  (existing pattern). `attacking` phase guards read `player.current_move` +
  `player.move_frame` thresholds. `defensive_status` guards read
  `player.invulnerable`.
- `pycats/systems/state_engine_sc.py`: `state` returns the active **`action`**
  leaf (map nested ids → the existing flat label set so `player.state` keeps
  working); add `defensive_status` property (reads the region leaf). Extend the
  `StateEngine` Protocol + `LegacyEngine` with a `defensive_status` derived from
  `player.invulnerable` so both backends expose it.
- Tests `tests/test_fighter_chart.py` (extend): `in_state` for nested/parallel ids;
  action-leaf label mapping matches the old labels for the non-attack states;
  both regions advance per tick.

### 4. Player: data-driven attack + move clock + defensive_status
- `pycats/entities/player.py`: add `current_move: MoveData|None`, `move_frame: int`,
  and a `defensive_status` property (delegates to engine). Load the fighter's
  `FighterData` in `__init__` via `load_fighter_data` (store hurtbox + moves).
- In `handle_actions`, on attack press set `current_move = moves["attack"]`,
  `move_frame = 0` (keep `attack_timer`/`done_attacking` for coarse legacy
  classification, set to `startup+active+recovery`). In `update`, advance
  `move_frame`; when it crosses into the active window, **spawn the circle hitbox**
  (see Task 5 Attack change); clear `current_move` when the move completes.
- Tests `tests/test_player_move.py`: move clock advances; hitbox spawns only during
  the active window; `defensive_status` is `intangible` during dodge.

### 5. Circle-based Attack + combat resolution
- `pycats/entities/attack.py`: Attack carries an absolute hitbox `Circle` (resolved
  from the move's facing-relative circle at spawn) + `damage`/`angle`, lives
  `active` frames; keep `rect` for rendering. (Knockback constants unchanged.)
- `pycats/systems/combat.py`: replace `atk.rect.colliderect(defender.rect)` with
  `circles_overlap(atk.circle, defender_hurtbox_circles)` (resolve the defender's
  `Hurtbox` circles from its rect+facing). Keep the `invulnerable`/`is_alive`/
  self-hit guards (now read via the formalized intangible status).
- Tests `tests/test_combat.py`: hit lands when circles overlap during active window;
  no hit during startup/recovery; intangible defender is skipped; self-hit excluded.

### 6. Extend the snapshot
- `pycats/sim/runner.py` `snapshot`: append `defensive_status` and `move_frame`
  (and the active hitbox's circle center/radius to the per-attack tuple) so golden
  snapshots capture both regions and the new combat geometry. Default the
  runner/`build_players` backend to `statechart`.

### 7. Flip the oracle to golden snapshots
- Add `tests/golden/` + a small helper (`tests/golden_util.py`) to record/compare
  golden per-frame snapshots; a `PYCATS_UPDATE_GOLDENS=1` path regenerates them.
- Convert `tests/test_parity.py` and `tests/test_full_match.py` from byte-identical
  *legacy-vs-statechart* into **statechart golden** tests over the same scripted
  timelines (default timeline, COMBAT_SCRIPT, and the captured full-match inputs).
  Record and commit the goldens.
- Keep `tests/test_runner.py` determinism checks (statechart vs itself).

### 8. Wire-up, benchmark, docs
- Default the live game/`watch.py`/`bench.py` to the `statechart` backend; keep a
  `--backend legacy` escape for the baseline.
- Re-run `bench.py` / `bench_render.py`; confirm the richer chart stays cheap; note
  results.
- Update `README.md` dev section and add the canonical plan to
  `docs/superpowers/plans/2026-06-21-pm-phase0.md` (copy of this plan) for
  subagent-driven execution.

## Verification
- `./.venv/bin/python -m pytest` on the new + converted tests (data, geometry,
  player-move, combat, fighter-chart, golden parity, full-match golden, runner).
- Record goldens once (`PYCATS_UPDATE_GOLDENS=1`), then assert stable on a re-run.
- `./.venv/bin/python bench.py --frames 5000` and
  `./.venv/bin/python bench_render.py` — confirm no perf regression.
- `./.venv/bin/python watch.py --match` — visually confirm the full battle still
  plays to a defeat under the statechart engine with the new circle attack.
